AWS ALB Routing Lab

Static pages designed to model path-based routing

Route: /

This homepage represents the root route of a simple Netflix-inspired learning site. The frontend stays static so you can focus on how an AWS Application Load Balancer will eventually route each URL path to the right target group and private EC2 instance.

How path-based routing will work

Concept Flow

One Application Load Balancer can inspect the requested URL path and forward traffic to the correct backend service for that route.

01

User Request

The browser sends a request to a path such as /movies.

→
02

Application Load Balancer

The public ALB receives the request and checks the URL path.

→
03

Listener Rule

The ALB matches the path rule, such as /movies*.

→
04

Target Group

The request is forwarded to the matching target group, such as tg-movies.

→
05

Private EC2 Instance

A private EC2 instance in that target group serves the route content.

→
06

Static Page Response

Nginx returns the correct static HTML page to the user.

Example request:

/movies → ALB listener rule /movies* → tg-movies → Movies EC2 server → Movies page
